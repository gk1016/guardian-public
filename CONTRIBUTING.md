# Contributing to Guardian Flight

Guardian Flight is built and maintained by Boring Tech Stuff LLC. Contributions are welcome.

## Reporting Bugs

Use the [Bug Report](https://github.com/gk1016/guardian-public/issues/new?template=bug_report.yml) template. Include:

- What happened and what you expected
- Steps to reproduce
- Your environment (OS, Docker version, browser)
- Relevant logs from `docker compose logs`

## Requesting Features

Use the [Feature Request](https://github.com/gk1016/guardian-public/issues/new?template=feature_request.yml) template. Describe the problem you're trying to solve, not just the solution you want.

## Discussion

For general questions, deployment help, or open-ended ideas, use [Discussions](https://github.com/gk1016/guardian-public/discussions) instead of opening an issue.

## Pull Requests

Before submitting a PR:

1. Open an issue or discussion first to describe what you want to change and why
2. Keep changes focused — one fix or feature per PR
3. Test your changes with a clean `./setup.sh` deployment
4. Make sure `docker compose ps` shows all services healthy

## Code of Conduct

Be respectful. This is a community project for Star Citizen organizations. Keep feedback constructive.
